# FocusBoard

FocusBoard — это учебный Flutter-проект про спокойную персональную продуктивность вокруг трёх ежедневных сценариев:

- аутентификация
- трекинг привычек
- mood check-in

Проект организован в feature-first стиле с разделением на `data`, `domain` и `presentation` слои. Сейчас приложение работает на локальном хранении и ручном роутинге, при этом domain API достаточно изолирован, чтобы позже можно было заменить инфраструктуру без полного переписывания приложения.

## Что умеет приложение

- проверяет auth state при запуске через отдельный auth flow
- позволяет пользователю войти, зарегистрироваться и выйти из аккаунта
- показывает список привычек с flow создания, редактирования, просмотра деталей и удаления
- отслеживает, выполнена ли привычка сегодня
- показывает небольшой daily summary блок
- позволяет отмечать настроение и смотреть историю mood check-in
- хранит данные локально, включая совместимый с web storage

## Архитектурные заметки

Сейчас в кодовой базе практикуются:

- feature-first организация
- state-driven обновление UI
- явные domain use cases
- repository abstraction
- локальные data source как источник истины
- route guards и typed route arguments

Presentation-слой сейчас построен на `ChangeNotifier`-based controllers.
Это сделано намеренно и честно названо в коде как `*Controller`, а не `Cubit`.

## Storage

Приложение использует небольшую локальную абстракцию хранения:

- file-based JSON на IO-платформах
- `localStorage` на web

Это прагматичный локальный persistence слой для текущего этапа, а не финальная production storage strategy.

## Что было изучено

Ниже — честный статус тем, которые изучались вокруг этого проекта.

| Тема         | Статус                                           | Комментарий                                                                                                                                                                               |
| ------------ | ------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `bloc`       | изучался концептуально, напрямую не используется | Проект ушёл от фейкового naming через "Cubit" и сейчас использует явный controller/state design. Следующим чистым шагом может быть миграция auth/habits/mood на настоящий `bloc`/`cubit`. |
| `freezed`    | пока не интегрирован                             | State-классы были сделаны более явными вручную. `freezed` был бы хорошим следующим шагом для immutable unions и copy helpers.                                                             |
| `auto_route` | пока не интегрирован                             | Навигация была приведена в более чистый вид через guards и typed arguments, но роутинг всё ещё ручной.                                                                                    |
| `dio`        | пока не интегрирован                             | У проекта пока нет удалённого API-слоя, поэтому `dio` не вводился.                                                                                                                        |
| `hive`       | пока не интегрирован                             | Локальное хранение сейчас реализовано через собственный JSON storage. `Hive` может быть следующим улучшением.                                                                             |
| `melos`      | пока не интегрирован                             | Репозиторий пока остаётся одним Flutter-приложением, а не multi-package workspace.                                                                                                        |
| `Firebase`   | пока не интегрирован                             | Auth сейчас локальный и персистентный, но это не Firebase Auth. Это осознанное текущее ограничение.                                                                                       |

## Что получилось потренировать на текущем этапе

Что реально было отработано в проекте:

- проектирование auth flow вместо `Future.delayed` на splash
- отделение UI от use cases и repositories
- построение list/detail/create/edit flow внутри feature
- работа с daily completion state и summary-вычислениями
- добавление локального persistence слоя без ломки domain API
- поддержка приложения как на IO-таргетах, так и на web
- уборка технического долга после архитектурных рефакторингов
